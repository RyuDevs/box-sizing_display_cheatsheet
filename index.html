box-sizing and display Cheatsheet

box-sizing CSS property

Property Value Description Syntax
box-sizing: content-box It's the default CSS box-sizing behavior. For Example if the width is set to 200px and the height 200px then the padding and border-size is added on top of the 200px. so if you add 20px padding then the box is 240px wide and the height is 240px. .example {box-sizing: content-box;}
border-box: This CSS box-sizing tells the browser that any border and padding values are within the width and height of the element. So if the elements width is 200px and the height is 200px. Now you add a border of 10px and padding of 50px then the height and width will remain 200px and the border and padding of the element is calculated of the element total width/height. Then the content of the element will only have left 80px height and 80px width. .example {box-sizing: border-box;}

display CSS property legacy values

Property Value Description Syntax
display: block The element generates a block element box, generating line breaks both before and after the element when in the normal flow. .example {display: block;}
inline The element generates one or more inline element boxes that do not generate line breaks before or after themselves. In normal flow, the next element will be on the same line if there is space .example {display: inline;}
inline-block The element generates a block element box that will be flowed with surrounding content as if it were a single inline box (behaving much like a replaced element would). .example {display: inline-block;}
flex The element behaves like a block element and lays out its content according to the flexbox model. .example {display: flex;}
inline-flex The element behaves like an inline element and lays out its content according to the flexbox model. It is equivalent to inline flex. .example {display: inline-flex;}
grid The element behaves like a block element and lays out its content according to the grid model. .example {display: grid;}
inline-grid The element behaves like an inline element and lays out its content according to the grid model. It is equivalent to inline grid. .example {display: inline-grid;}
flow-root The element generates a block element box that establishes a new block formatting context, defining where the formatting root lies. .example {display: flow-root;}
none Turns off the display of an element so that it has no effect on layout (the document is rendered as though the element did not exist). All descendant elements also have their display turned off. To have an element take up the space that it would normally take, but without actually rendering anything, use the visibility property instead. .example {display: none;}
